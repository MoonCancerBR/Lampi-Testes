# Automação de Login - Mobak (Front-end)

Este documento descreve os requisitos e objetivos para a implementação da automação de interface utilizando o **Robot Framework**.

## 🎯 Objetivos

* **Automação do Navegador:** O teste deve abrir o navegador automaticamente.
* **Acesso:** O teste deve acessar a tela de login do Mobak.
* **Interação:** O teste deve preencher os campos de usuário e senha.
* **Ação:** O teste deve clicar no botão de login.
* **Validação:** O teste deve validar que o login foi realizado com sucesso (ex.: redirecionamento ou presença de elemento da home/dashboard).
* **Autonomia:** O teste deve rodar sem intervenção manual.
* **Organização:** O código deve ser organizado com uso de *keywords* reutilizáveis.
* **Metodologia:** A solução deve seguir os princípios de **BDD** utilizando **Gherkin**.
* **Arquitetura:** As *keywords* devem ser granularizadas (baixo acoplamento e alta reutilização).
* **Padronização:** O projeto deve estar totalmente em **inglês**.

## 📦 Entregável (POC)

Entregar a automação do fluxo de login atendendo a todos os objetivos especificados acima.
